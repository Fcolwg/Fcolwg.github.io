BO 강의용 앱 구조

교수자와 학생이 같은 링크에서 보는 단일 정적 웹앱 구조입니다.

단일 화면 구성

  • 왼쪽: 차원, 평가 대상 함수 f, 획득함수, sampling, 반복수 제어
  • 가운데: 1D/2D/3D 평가 대상 함수 f와 BO/sampling 관측점
  • 가운데 아래: 1D GP posterior
  • 오른쪽: BO best, sampling best, 계산별 점수, best score 그래프

계산 흐름

  1. target function f 수식 입력 또는 1D 직접 그리기
  2. initial sampling 생성
  3. 관측점 기반 GP posterior 추정
  4. EI, PI, UCB 또는 사용자 정의 획득함수 계산
  5. 다음 후보점 선택
  6. history snapshot 저장
  7. iteration별 plot과 replay 제공

시각화 범위

  • 1D: 평가 대상 함수 f, BO 관측점, sampling 관측점, 다음 후보점, GP posterior, 점수 그래프
  • 2D: contour/heatmap, BO 관측점, sampling 관측점, 다음 후보점
  • 3D 입력: 3D scatter와 z=0.5 slice view, 다음 후보점

구현 우선순위

  1. 1D GP posterior 설명 강화
  2. 2D 직접 그리기 heatmap 편집 추가
  3. 배포 URL 확정 후 QR 코드 삽입
